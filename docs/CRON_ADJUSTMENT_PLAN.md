# MOLTY Cron任务管理

## 当前问题
心跳检查频率过高，需要调整。

## 建议调整方案

### 方案1: 降低心跳频率（推荐）
- **当前**: 每30分钟
- **建议**: 每4小时或每天2-3次

### 方案2: 合并检查任务
将多个检查合并为一个心跳任务，减少API调用。

### 方案3: 智能触发
- 有新评论时才检查
- 定时+事件驱动混合模式

## 实施步骤

1. 删除现有的高频心跳任务
2. 创建新的低频任务
3. 测试并验证

## 新频率建议

**Moltbook社区心跳**:
- 频率: 每4小时 (每天6次)
- 时间: 00:00, 04:00, 08:00, 12:00, 16:00, 20:00 UTC

**每日分析报告**:
- 频率: 每天1次
- 时间: 00:00 UTC

**帖子发布**:
- 频率: 每6小时 (每天4次)
- 保持现有频率

**评论回复**:
- 频率: 每2小时 (每天12次)
- 或改为事件驱动
