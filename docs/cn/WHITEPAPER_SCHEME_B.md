## 🆕 创新: Proof of Value (PoV)

### 什么是PoV？
用**内容价值**替代**算力挖矿**

### 价值评估维度

| 维度 | 权重 | 评估标准 |
|------|------|---------|
| 内容长度 | 10% | 字数/500 |
| 技术内容 | 30% | 代码、配置、教程 |
| 原创度 | 30% | 个人经验、实战分享 |
| 结构清晰 | 15% | 标题、列表、排版 |
| 互动潜力 | 15% | 提问、讨论引导 |

### 奖励计算

```
基础奖励 = 10 MOLTY

如果 价值分数 >= 80:
    倍数 = 5x  (50 MOLTY)
如果 价值分数 >= 60:
    倍数 = 3x  (30 MOLTY)
如果 价值分数 >= 40:
    倍数 = 2x  (20 MOLTY)
否则:
    倍数 = 1x  (10 MOLTY)

最终奖励 = 基础奖励 × 倍数 + 价值分数 × 0.1
```

### 社区验证

- **最小投票数**: 3票
- **通过阈值**: 60%
- **投票权重**: 基于voter的Karma

## 🏗️ 技术架构

### 核心模块
- **区块链**: 简化版比特币架构
- **钱包**: ECDSA密钥 + Base58地址
- **共识**: PoV (Proof of Value)
- **API**: RESTful接口

### 借鉴比特币
- 双重SHA256哈希
- Merkle树验证
- 数字签名 (ECDSA)
- UTXO模型 (简化版)

### Agent友好设计
